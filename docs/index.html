This website provides instructions on how to validate your device works with Linux, with a focus on laptops, and how to safely switch to it.

Most devices are supported by Linux releases. In some rare cases, some peculiar features may not fully be supported (e.g. fingerprint sensors, ...). This page provides instructions on how to safely test Linux on your device and eventually switch.

Please note this process is generally simple, but will need patience, time and some USB devices (1 minimum if you already have data backups, otherwise 2 or more depending on your data).

Time-wise, if this is your first time following this process this, it could take between a couple of hours to a full day (depending on your data backup procedure).

To start, a bit of context:

  • Linux is the kernel of the operating system, which is the central part handling all communications between hardware and software
  • Linux distros (distributions) are full operating sysetems (like Windows and MacOS) which can be installed on a device
All operating systems have this separation, in most cases it is just hidden in a monolytic system.

We will proceed in order to:

  1. Check for any blockers
  2. Backup all your data
  3. Choose a Linux operating system (distro)
  4. Create a live USB
  5. Enable your device to start the live USB
  6. Test the operating system without installing
  7. Install the operating system

Instructions are mainly targeted for Windows-based devices, but are generally applicable to other devices like Apple ones with some caveats regarding step 2.

Please remember this website and its contributors do not provide any direct support for this operation. If you know people with experience regarding software development or generally tech-enthusiasts, it is possible they may be able to provide a bit of support. Look also for "Linux Install Party" events and "Linux User Group".

0. Check for any blockers

Before any step, we would reccomend to verify if any of the below things are blockers:

  • Microsoft Office does not work on Linux, but it can be replaced by the free LibreOffice. LibreOffice can open and edit Microsoft Office documents, but may provide some slight dicrepancies if the same document is opened frenently in both softwares. This can be reduced by using the open standard ODT format instead of the Microsoft DOCX and similar format.
  • Adobe Photoshop and similar instruments may not work fully. There are ways to get them mostly working via compatibility layers, but they can also fully be replaced by other tools like GIMP.
  • Cloud backup software by Microsoft and Google (OneDrive, Google Drive) does not officially work with Linux. This means that there is not an official way to connect to the systems, but only software by volunteers which provides most of the standard features. Other cloud backup may and will work.
  • If you use accessibility devices or external components like drawboards or sensors and other non-common systems, please refer to online searches to identify if your specific device will work - there are usually some resources or user feedback available.
The FAQ section of this website provides more information regarding the previous steps and what can be done to limit any issues.

If you are unsure about anything, you can actually test most specialized software for Linux (e.g. LibreOffice instead of Microsoft Office, GIMP instead of Adobe Photoshop, ...) on Windows. This may help you choose to switch or not.

1. Backup all your data

If you want to fully switch to Linux, please make sure you copy all your files and data to a separate storage.

We generally reccomend using a USB device with enough storage, or using an online cloud backup solution.

The FAQ section provides more information on what cloud backup solutions support Linux offically.

You should also note what software you use more commonly, especially Office, e-mail and browsers, to verify if the same software is available on Linux or which variants to target. Remember you can usually test the software variants on Windows too before switching.

You should also ensure your Windows license will be available for use later on, in case you ever need to revert back to Windows.. You can do so by checking the "Settings" -> "System Info" panel on Windows and the instructions at the official website. If Windows was the default operating system provided to the device when sold by the original producer, then the Windows license is most likely stored and linked to the device and Windows can be reinstalled as needed without noting it down - but always double check this.

There are many more distros available, so you may choose any which provides the experience you are looking for.

You will need to download the ISO file for the respective distro you choose, looking for the desktop version.

A specal note for Apple Mac devices using the M* processors: the procedures at this page are relevant still, but you should especially check Ashai Linux

If your device is from before 2006, it may have an older architecture for its processor which needs a special Linux distro. That is the 32bit (i386) architecture.

If your device is using Windows 11, then it must have the 64bit architecture and you may ignore this step.

You can verify your device architecture by checking the model on the label at the bottom of your laptop or looking on the Windows "Settings" -> "System Info" under "System Type".

If your device has the 32bit architecture, you should check Puppy Linux.

This step is optional as we will validate your device works correctly before installing anything

Using the model name,you can double check via online searches or specialized websites (e.g. Linux Hardware) if you should expect any issues on your device using Linux (like specific sensors not working, like fingerprint ones or similar).

3. Create a live USB

We will now prepare the live install and test USB for you to use in later steps to check your chosen distro works as expected and eventually install it.

We need:

  1. 1 USB device with at least 8GB of storage
  2. ISO image of the chosen distro
  3. Rufus software (or others if you know about them) to write the image to the USB

Start by connecting the USB to your device, and checking it does not contain any data you need to keep - everything on it will be deleted. Then download Rufus and open it, select your ISO image via the interface and the USB image to write to, and start the process.

The process to write may take a few minutes (generally less than 20 minutes), and will provide some warnings to ensure you know all data inside the USB will be lost. Do not disconnect the USB while the process is ongoing, and if there is any error retry by connecting the USB to another port of your device.

Once Rufus prompts you with a success message, you are ready for the next step.

If you fear that once you install Linux you cannot switch back to Windows, you can just prepare a live USB for Windows too and eventually follow steps 4 & 5 to revert to Windows even if you install Linux. You must use a separate USB device from the ones used for backup and/or the Linux live USB. Please note you must ensure the Windows license will work correctly following instructions at step 1. To prepare the USB, use the official Windows utility or download the respective ISO image.

If you need to create a live USB from Linux, we reccomend to check the instructions from the Ubuntu distribution wiki .

5. Enable your device to start the live USB

This step is a bit more technical than the previous ones, as it requires to open the device boot menu on the BIOS and allow the live USB to be used as the startup point for your device operating system.

Do not fear, as we will provide all relevant information and the procedure is safe.

In order:

  1. Shutdown your device
  2. Connect your USB device to the powered-off device
  3. Identify the keys "Esc", "F2", "F12" and "Del" of your keyboard
    • Device and manufacturer may have variants, but one of the above generally work
    • In the next step, you will need to click one or all of them repeatedly as soon as you power on the device. If you click 1 only and it is the wrong one, you will just get Windows again and need to try another by restarting. Clicking them all in parallel does not cause issues.
  4. Power on the device and immediately start clicking the keys repeatedly until a different interface loads
  5. Device and manufacturer will have different interfaces, but without any urgency:
    1. Look for a "Boot" menu and "Boot Device Order/Priority" setting (names may vary a bit)
    2. Follow the interface instructions to move the USB device before the hard disk
    3. Look also for "UEFI Secure Boot Certificate" setting in the same section
    4. If it contains a "Windows" or "Microsoft" reference on it, look if there is another option for "Secure Boot Supported OS"
    5. Ensure you did not change anything else by error - if you are in doubt, you can always just exit without saving and starting this step by scratch
    6. Save and exit
  6. This will automatically restart the device and startup from the live USB for the next step

6. Test the operating system without installing

By following the previous step, the next device restart will automatically use the inserted live USB to startup. This will load the custom UI of the chosen distro on your device. Never remove the live USB from your device unless explicitly said by the device itself on the display.

Each distro will have slightly different interface, but you should generally see a language selector and options to "Try" or "Install" the distro.

At this step, we want for you to experiment a bit with the chosen distro to verify you like it. Please select "Try" and once the system loads, feel free to play around. You should be able to connect to the internet and browse, and start using base applications available - installing new ones would not really be doable until you oficially switch to Linux, but feel free to look on the Software/Application center for examples, and more are available online!

If you have any peculiar systems and components in your device you need for your daily life, you should also check they work correctly on this system, and eventually find some time to check online for any solutions.

Once you are satisfied by your testing and want to proceed, go to the next step without restarting! If you want to go back to Windows, then find the shutdown menu and follow instructions.

7. Install the operating system

You have chosen to install the Linux distribution you tested above. Now, you should be able to find an option on the start menu or desktop of the current system to install the distro.

There is one main choice to do at this step which requires instruction: the choice of keeping Windows and add Linux in dual boot, or install Linux only. For non-advanced users, we reccomend to install Linux only.

After this choice, for the most part you just need to follow instructions on the UI and proceed with setup of your account.

You may want to keep both Windows and Linux on your device.

If so, please note that you may always use 1 only at a time, and that you will need to keep some storage for both systems - you will practically reduce the available storage for your device. You will also be prompted at every device startup if you want to use the Linux distro or Windows.

The procedure to do so is to simply use the installer UI and when prompted on how to install the Linux distro, choose a "dual-boot" option or the advanced option.

The procedure may be easier on some distro than in others - you will need create a partition (limited storage of a specified number of GB) from the existing Windows partition, and then select the new partition as the place where Linux should be installed.
